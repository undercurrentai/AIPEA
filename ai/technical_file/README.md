# Technical File (Annex IV alignment)

This directory stores the technical documentation required to demonstrate compliance for high-risk AI systems and supports transparency for other systems.

Structure:
- `instructions-for-use.md` — deployer guidance (purpose, limitations, installation, oversight).
- `testing/` — TEVV results: accuracy, robustness, cybersecurity.
- `logs/` — mapping of system logs to required record-keeping.
- `conformity/` — standards used, risk mgmt approach, declarations and checklists.

Keep this folder up to date prior to release and during operations.
